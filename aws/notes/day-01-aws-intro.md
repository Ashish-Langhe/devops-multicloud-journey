## 📌 What is Cloud Computing?

```
Traditional (On-Premises)          Cloud Computing
─────────────────────────          ───────────────────────────
Buy physical servers               Rent servers over the internet
Manage electricity, cooling        Provider manages everything
High upfront cost                  Pay only for what you use
Scalability is hard                Scale up/down in minutes
High availability is expensive     Built-in HA options
```

> 💡 **Simple definition:** Cloud computing = connecting to a cloud provider over the internet and deploying required resources from their infrastructure.

---

## ☁️ What is AWS?

- **AWS = Amazon Web Services**
- A **cloud platform** where you get IT resources (servers, storage, databases, networking) on a **rental basis**
- First cloud in the market → launched in **2006**
- Introduced the **pay-as-you-go** model to the world

## 🏗️ On-Premises vs Cloud — Real Example (Wipro)

### On-Premises (Wipro's own server room)
```
Wipro needs servers →
  Must manage:
  ├── Electricity
  ├── Manpower
  ├── Ventilation / Cooling
  ├── Backup power
  ├── High Availability setup
  └── Scalability planning
```

### Cloud (AWS)
```
Wipro needs servers →
  Connect to AWS →
  ├── Pick region
  ├── Pick instance type
  ├── Pay per hour
  └── AWS manages all physical infrastructure ✅
```

## 👨‍💻 Role of a DevOps Engineer

> DevOps = **Responsible for deploying applications to servers for end-users**

```
Developer writes code
        ↓
DevOps Engineer takes the code
        ↓
Deploys it to a server (on-prem or cloud)
        ↓
End users access the application
```

DevOps can deploy to:
- **On-premises server** — company's own data centre
- **Cloud server** — AWS, Azure, GCP
